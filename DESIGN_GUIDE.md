## 设计原则与理论基础

### 1. 颜色对比度原则（A与B的对比要求）

**核心要求：A（主要文字颜色）与B（主要底色）必须对比鲜明**

#### WCAG 对比度标准
- **最小对比度**：4.5:1（AA级标准，适用于正常文字）
- **推荐对比度**：7:1（AAA级标准，适用于重要文字）
- **计算公式**：对比度 = (L1 + 0.05) / (L2 + 0.05)
  - L1 = 较亮颜色的相对亮度
  - L2 = 较暗颜色的相对亮度

#### 对比度检查方法
1. **亮度对比**：确保A和B在亮度上有明显差异
   - 深色B → 使用浅色A（白色、浅灰、浅黄等）
   - 浅色B → 使用深色A（黑色、深灰、深蓝等）

2. **色相对比**：除了亮度，色相差异也能增强对比
   - 互补色对比（如红-青、蓝-橙）
   - 冷暖色对比（如暖色B配冷色A）

3. **实际应用建议**
   - 深色主题（B为深色）：A选择 #FFFFFF、#F5F5F5、#FFE4C8 等
   - 浅色主题（B为浅色）：A选择 #000000、#1A1A1A、#2C2C2C 等
   - 中等亮度B：根据具体情况选择，确保对比度≥4.5:1

### 2. 色带配色理论（C-F四色搭配方法）

**核心要求：C、D、E、F四色可以是渐变色或和谐的不同颜色，需适当呼应主题色B**

#### 方案一：渐变配色方案

**特点**：四色形成平滑过渡，视觉统一性强

**设计方法**：
1. **基于B色的渐变**
   - 从B色出发，向相邻色相方向渐变
   - 保持相同或相近的饱和度，调整亮度
   - 示例：B为深红(#8B1A1A) → C(#FF8C42) → D(#FFB884) → E(#FFCFA8) → F(#FFE4C8)

2. **亮度渐变**
   - 从深到浅或从浅到深
   - 保持色相基本一致
   - 适用于单色或近似色系

3. **饱和度渐变**
   - 从高饱和度到低饱和度
   - 适用于需要柔和过渡的场景

**渐变方向选择**：
- **暖色渐变**：红→橙→黄→米色
- **冷色渐变**：深蓝→蓝→浅蓝→青
- **中性渐变**：深灰→中灰→浅灰→米白

#### 方案二：和谐配色方案

**特点**：四色在色相环上形成和谐关系，色彩丰富但统一

**设计方法**：
1. **类似色配色**（Analogous）
   - 选择色相环上相邻的3-4个颜色
   - 如：黄绿→绿→青绿→青
   - 视觉和谐，适合自然主题

2. **互补色配色**（Complementary）
   - 选择B色的互补色及其变化
   - 如：B为红色，C-F选择青色系的变化
   - 对比强烈，适合需要强调的场景

3. **三角配色**（Triadic）
   - 在色相环上形成等边三角形
   - 如：红、黄、蓝及其变化
   - 色彩丰富，需要控制饱和度

4. **分裂互补色**（Split Complementary）
   - 选择B色的互补色两侧的颜色
   - 如：B为红色，C-F选择黄绿和蓝绿的变化
   - 既有对比又保持和谐

**和谐配色注意事项**：
- 保持至少一种颜色与B色呼应（色相相近或互补）
- 控制饱和度，避免过于鲜艳导致视觉疲劳
- 确保C-F之间有足够的区分度，避免混淆

### 3. 主题色呼应原则（色带与B的关系）

**核心要求：色带要适当呼应主题颜色B，主题色B也会充当色带区域的文字颜色**

#### 呼应方式

1. **色相呼应**
   - C-F中至少1-2个颜色与B色相相近
   - 如：B为深红，C选择橙红，D选择橙，E和F选择浅橙/米色

2. **明度呼应**
   - C-F的整体明度与B形成呼应
   - 深色B → C-F整体偏深（但比B浅）
   - 浅色B → C-F整体偏浅（但比B深）

3. **饱和度呼应**
   - C-F的饱和度与B形成呼应
   - 高饱和度B → C-F保持较高饱和度
   - 低饱和度B → C-F使用柔和的低饱和度色

#### 文字颜色应用

**重要提示**：主题色B会充当色带区域的文字颜色

这意味着：
- 如果B是深色，在浅色色带（C-F）上显示效果良好
- 如果B是浅色，在深色色带（C-F）上显示效果良好
- **设计时需考虑B色在C-F各色上的可读性**

**可读性检查**：
- B色在C上的对比度 ≥ 3:1（最低要求）
- B色在D上的对比度 ≥ 3:1
- B色在E上的对比度 ≥ 3:1
- B色在F上的对比度 ≥ 3:1

## 颜色选择指导

### A（主要文字颜色）选择指南

#### 选择原则

1. **与B形成强烈对比**
   - 深色B → 浅色A（白色、浅灰、浅米色）
   - 浅色B → 深色A（黑色、深灰、深蓝）

2. **考虑整体风格**
   - 现代简约：黑白对比
   - 复古风格：米色、暖灰
   - 科技风格：冷色系（青、蓝）

3. **避免的问题**
   - ❌ A与B对比度不足（<4.5:1）
   - ❌ A色过于鲜艳，抢夺视觉焦点
   - ❌ A色与C-F色带颜色过于接近

#### 推荐配色

| B色类型 | 推荐A色 | 示例 |
|---------|---------|------|
| 深色（亮度<50） | #FFFFFF, #F5F5F5, #FFE4C8 | 深红B配白色A |
| 中等深度（50-150） | #000000, #1A1A1A, #2C2C2C | 中蓝B配黑色A |
| 浅色（亮度>200） | #000000, #1A1A1A, #2C2C2C | 浅灰B配黑色A |

### B（主要底色）选择指南

#### 选择原则

1. **作为设计主题的基础**
   - B色决定了整体色调和风格
   - 应与年份主题的情感色彩匹配

2. **考虑文字可读性**
   - 确保A色在B色上清晰可读
   - 确保B色在C-F色带上清晰可读（作为文字颜色）

3. **情感表达**
   - 红色系：热情、力量、历史事件
   - 蓝色系：科技、未来、冷静
   - 绿色系：自然、生机、环保
   - 黄色系：活力、阳光、希望
   - 紫色系：神秘、创意、艺术

#### 推荐色相

| 主题类型 | 推荐B色系 | 示例 |
|----------|-----------|------|
| 历史事件 | 深红、深蓝、深绿 | #8B1A1A, #1A3A5A, #2D5016 |
| 科技品牌 | 深蓝、深灰、深紫 | #1E3A5F, #2C2C2C, #4A148C |
| ACG角色 | 根据角色主色调 | 角色代表色 |
| 自然主题 | 深绿、深棕、深蓝 | #2D5016, #5D4037, #1A3A5A |

### C-F（四色色带）配色方案

#### 渐变方案设计流程

**步骤1：确定起点和终点**
- 起点（C）：与B色呼应，但比B浅
- 终点（F）：最浅的颜色，接近米色或浅灰

**步骤2：计算中间色**
- D = C与F的中点色
- E = D与F的中点色

**步骤3：调整饱和度**
- 确保过渡自然
- 避免某个颜色过于突兀

**示例：深红主题渐变**
```
B: #8B1A1A (深红)
C: #FF8C42 (深橙) - 与B呼应，暖色系
D: #FFB884 (中橙) - 过渡色
E: #FFCFA8 (浅橙) - 过渡色
F: #FFE4C8 (米色) - 最浅色
```

#### 和谐配色方案设计流程

**步骤1：确定配色类型**
- 根据主题选择类似色、互补色等

**步骤2：选择色相**
- 在色相环上选择4个位置
- 确保至少1个与B呼应

**步骤3：调整明度和饱和度**
- 保持整体和谐
- 确保B色在各色上可读

**示例：蓝色科技主题**
```
B: #1E3A5F (深蓝)
C: #4A90E2 (亮蓝) - 与B同色相
D: #7BB3F0 (浅蓝) - 过渡
E: #A8D0F5 (更浅蓝) - 过渡
F: #D4E8F8 (极浅蓝) - 最浅
```

#### 对比度检查方法

**工具推荐**：
- 在线工具：WebAIM Contrast Checker
- 设计软件：Figma、Adobe Color
- 代码计算：使用WCAG公式

**检查清单**：
- [ ] A与B对比度 ≥ 4.5:1
- [ ] B与C对比度 ≥ 3:1（B作为文字）
- [ ] B与D对比度 ≥ 3:1
- [ ] B与E对比度 ≥ 3:1
- [ ] B与F对比度 ≥ 3:1
- [ ] C、D、E、F之间有足够区分度

## 年份主题设计方法

### 年份主题选择策略

#### 1. 历史事件主题

**选择原则**：
- 选择具有重大意义的历史事件年份
- 考虑事件的情感色彩和视觉特征
- 关键词应简洁有力，体现事件核心

**设计思路**：
- **战争/革命**：深红、深灰，关键词如"Victory"、"Freedom"
- **科技突破**：深蓝、深紫，关键词如"Apollo"、"Internet"
- **文化运动**：根据文化特色选择，如"Woodstock"、"Beatles"

**示例年份**：
- 1945：二战结束 → "Victory"、"Peace"
- 1969：登月 → "Apollo"、"Moon"
- 1989：互联网诞生 → "Internet"、"Web"

#### 2. 品牌诞生年份主题

**选择原则**：
- 选择知名品牌的创立年份
- 使用品牌标志色或经典配色
- 关键词可以是品牌名或品牌理念

**设计思路**：
- **科技品牌**：使用品牌主色调，如苹果的银灰、微软的蓝
- **汽车品牌**：使用品牌经典色，如法拉利红、保时捷银
- **时尚品牌**：使用品牌代表色，如香奈儿的黑白、LV的棕

**示例年份**：
- 1976：Apple成立 → "Apple"、"Think Different"
- 1963：Lamborghini成立 → "Lamborghini"、"Speed"

#### 3. ACG角色诞生年份主题

**选择原则**：
- 选择经典动漫、游戏角色的首次出现年份
- 使用角色的标志色和特征色
- 关键词可以是角色名或角色特征

**设计思路**：
- **动漫角色**：使用角色发色、服装色、代表色
- **游戏角色**：使用角色主题色、技能色
- **超级英雄**：使用英雄制服色、标志色

**示例年份**：
- 1979：机动战士高达 → "Gundam"、"Mobile Suit"
- 1986：塞尔达传说 → "Zelda"、"Legend"
- 1997：哈利波特 → "Potter"、"Magic"

### 关键词提取方法

#### 关键词选择原则

1. **简洁有力**
   - 1-2个单词最佳
   - 避免过长或复杂词汇

2. **主题相关**
   - 直接关联年份事件/品牌/角色
   - 体现核心概念或情感

3. **视觉可读**
   - 字母形状清晰
   - 避免容易混淆的字母组合

#### 关键词类型

**类型1：直接命名**
- 品牌名：Apple、Nike、Sony
- 角色名：Gundam、Zelda、Mario
- 事件名：Apollo、Victory、Peace

**类型2：概念表达**
- 情感：Freedom、Hope、Dream
- 理念：Innovation、Speed、Magic
- 特征：Classic、Modern、Retro

**类型3：组合表达**
- 年份 + 关键词：如"1945 Victory"
- 品牌 + 理念：如"Apple Innovation"

### 文字配置最佳实践

#### 配置规则

1. **年份（4位数字）**
   - 必须4位数字
   - 使用斜体字体（Stapel Narrow Extra Bold Italic）
   - 位置：(780, 400)
   - 字号：18px（根据canvas缩放）

2. **英文单词第一处（回车分隔）**
   - 位置：(780, 418)
   - 字体：Stapel Narrow Extra Bold Italic
   - 字号：11px
   - 首字母大写

3. **英文单词第二处（空格分隔，非必填）**
   - 位置：(467, 515)
   - 字体：Fresty Personal Use Only
   - 字号：22px
   - 首字母大写
   - 优先使用wordSpace，如果没有则使用word

#### 设计建议

- **单关键词方案**：只使用word，简洁明了
- **双关键词方案**：word + wordSpace，层次丰富
- **避免过度**：不要使用过多文字，保持简洁

- B-F: 7.2:1 ✅

- 年份：1945

## 最佳实践与常见错误

### 配色常见问题

#### 问题1：A与B对比度不足

**症状**：
- 文字模糊不清
- 视觉疲劳
- 可读性差

**解决方案**：
- 检查对比度，确保≥4.5:1
- 深色B配浅色A，浅色B配深色A
- 使用在线对比度检查工具验证

#### 问题2：C-F色带颜色过于接近

**症状**：
- 色带区分度低
- 视觉层次不清晰
- 整体单调

**解决方案**：
- 确保相邻颜色有明显差异
- 使用渐变时保持足够的步进
- 和谐配色时选择色相环上不同位置

#### 问题3：B色在C-F上不可读

**症状**：
- B色作为文字颜色时模糊
- 色带区域文字难以识别

**解决方案**：
- 设计时考虑B色在各色带上的可读性
- 确保B与C-F各色对比度≥3:1
- 如果B是深色，C-F选择浅色；反之亦然

#### 问题4：色带与B色缺乏呼应

**症状**：
- 整体不协调
- 主题色孤立
- 视觉割裂

**解决方案**：
- 确保C-F中至少1-2个颜色与B呼应
- 使用色相、明度或饱和度呼应
- 渐变方案从B色出发

### 避免的错误

#### 错误1：过度使用鲜艳颜色

**问题**：
- 视觉疲劳
- 抢夺焦点
- 不专业

**正确做法**：
- 控制饱和度
- 使用柔和的中间色
- 只在关键位置使用高饱和度

#### 错误2：忽略文字可读性

**问题**：
- 文字难以识别
- 用户体验差
- 设计失败

**正确做法**：
- 始终检查对比度
- 考虑B色在C-F上的可读性
- 使用对比度检查工具

#### 错误3：年份主题与配色不匹配

**问题**：
- 主题表达不清晰
- 情感传达错误
- 设计不协调

**正确做法**：
- 根据主题选择合适色相
- 考虑主题的情感色彩
- 确保配色支持主题表达

#### 错误4：关键词选择不当

**问题**：
- 关键词过长或复杂
- 与主题不相关
- 视觉混乱

**正确做法**：
- 选择1-2个简洁关键词
- 确保与年份主题相关
- 考虑字母形状和可读性

### 设计检查清单

在完成设计前，请检查以下项目：
